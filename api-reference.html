    Plausible.Ingestion.Counters

    This is instrumentation necessary for keeping track of per-domain internal metrics. Due to metric labels cardinality (domain x metric_name), these statistics are not suitable for prometheus/grafana exposure, hence an internal storage is used.

    Plausible.Ingestion.Counters.Buffer

    A buffer aggregating counters for internal metrics, within 10 seconds time buckets.

    Plausible.Ingestion.Counters.Record

    Clickhouse schema for storing ingest counter metrics

    Plausible.Ingestion.Counters.TelemetryHandler

    Susbcribes to telemetry events emitted by Plausible.Ingestion.Event. Every time a request derived event is either dispatched to clickhouse or dropped, a telemetry event is emitted respectively. That event is captured here, its metadata is extracted and sent for internal stats aggregation via Counters.Buffer interface.

    Plausible.Ingestion.Event

    This module exposes the build_and_buffer/1 function capable of turning %Plausible.Ingestion.Request{} into a series of events that in turn are uniformly either buffered in batches (to Clickhouse) or dropped (e.g. due to spam blocklist) from the processing pipeline.

    Plausible.Ingestion.Request

    The %Plausible.Ingestion.Request{} struct stores all needed fields to create an event downstream. Pre-eliminary validation is made to detect user errors early.

    Plausible.OpenTelemetry.Sampler
